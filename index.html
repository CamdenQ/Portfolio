DEKKER: An MTG Deck Builder

DEKKER is a deck-building app for the popular trading card game Magic: the Gathering.

With DEKKER, the user can browse and sort the cards available, create decks from those cards, and add cards to or remove cards from existing decks.

DEKKER is a completely solo endeavor. It's still a work in progress, but I'm pretty pleased with this first stage. This version took roughly two weeks to make, mostly due to it being the first full-stack application I had built on my own.

Planned improvements include incorporating user registration and authentication, hooking in the official MTG API rather than relying on a custom cards database, adding even more filter options, and improving the UI and UX.

A long-term goal is to eventually add other card games to DEKKER so that it can be a central resource for all of your trading card game deck building needs

Technologies used (not exhaustive):

  • Javascript
  • HTML 5
  • CSS 3
  • React.js
  • Express.js
  • Node.js
  • Knex.js
  • PostgreSQL

Client GitHub: https://github.com/CamdenQ/dekker-capstone-client

API GitHub: https://github.com/CamdenQ/dekker-capstone-api

BuddyUp: Find local activities and people

BuddyUp is an app designed to help users link up with people in their area for specific activities. The app is primarily designed with users in mind who may not know anybody locally but still want to have some company when they go out. With BuddyUp, all the user needs to do is post their activity or find one that's already been posted in their area.

This is a proof of concept, so many concerns, especially security conserns beyond basic authentication, are left unhandled.

I worked on a team of three for this project. I was responsible for 90% of the API. The authentication middleware and endpoints were plugged in by another team member from a boilerplate we all had access to, and he did the API testing as well. Every other part of the API was built by me alone. I also had a hand in several React components of the client (including the activity signup component and the list of user signups on the account page), though most of the front-end and 100% of the design were another team member's work.

Spaced Repetition Learning

This app was another proof of concept, this time built as part of a two-man team. It was built in about a day with very specific requirements, including that it use a spaced repetition algorithm to assist the user learning words in a foreign language.

The idea was to simulate working on an existing code-base to implement features. The database was already built, and we had to design and implement the algoritm on the back end as well as build the front end.

The work was completed in about a day's worth of man hours.
